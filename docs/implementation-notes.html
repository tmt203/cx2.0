Routing & Page Resolver

Route strategy

1

src/app/page.tsx

Trang gốc redirect thẳng về /dashboard.

2

(default)/layout.tsx

Bọc toàn bộ page app bằng AppLayoutTemplate để có sidebar.

3

(default)/[...slug]/page.tsx

Route động, dynamic = "force-dynamic", truyền slug vào PageResolver.

4

PageResolver.tsx

Normalize route, thử các route candidate, gọi Directus ui_pages, rồi dùng trailing segments để phân biệt list, create /new và edit /:id.

Quyết định render

page_type Điều kiện Component
collection Cần collection_key hoặc collection từ trailing segment CollectionPage hoặc CollectionRecordPage
custom component_key hoặc key có trong registry Hiện có dashboard -> DashboardPage
Không khớp Không có page, disabled, thiếu key/component notFound()

Directus Contract

ui_pages

Field Vai trò trong UI
route Được PageResolver dùng để match URL.
page_type Chọn giữa page collection hoặc custom component.
collection_key Tên Directus collection dùng cho CollectionPage.
component_key Key tra trong customPageRegistry.
is_enabled False thì route trả 404.

ui_sidebar_items

Field Vai trò trong sidebar
type group, collapse, item quyết định cấu trúc menu.
parent_id Dựng cây cha-con cho group/collapse/sub item.
page_id.route Route thật cho link sidebar.
translations Label theo locale hiện tại.
sort_order Sắp xếp item trong cùng cấp.

translations

Directus translations được preload qua apiGetTranslations() và lưu trong appStore.translations. Dynamic form dùng dữ liệu này để resolve option label có dạng $t:key theo locale hiện tại.

Component System

Atoms

Các phần tử nhỏ: Icon, Breadcrumb, TableHeader, TableBody, toggles, dropdown nhỏ, search input.

Molecules

Tổ hợp input/action: Button, InputForm, SelectForm, Modal, Pagination, SidebarMenu.

Organisms

Cụm UI lớn: Sidebar, DataTable, Filter. Đây là nơi workflow và trạng thái tương tác bắt đầu rõ hơn.

Templates

AppLayoutTemplate chịu trách nhiệm dữ liệu nền của app và sidebar. DefaultPageLayout cung cấp header, breadcrumb, action area và vùng nội dung scroll.

Pages

PageResolver là lớp điều phối. Page cụ thể gồm DashboardPage, CollectionPage và CollectionRecordPage; custom page mới cần đăng ký trong customPageRegistry.ts.

Collection List & Record CRUD

Luồng dữ liệu

A

Metadata từ store

Fields, collections và translations được load một lần ở layout. CollectionPage và CollectionRecordPage lọc fields theo collection hiện tại.

B

Dựng columns

Field type được map sang ColumnType; label lấy từ meta.translations; hidden field bị ẩn.

C

Load records

apiGetItems gọi /items/{collection} với page, limit, sort và filter params. Search state vẫn còn trong code nhưng input search đang được comment ở UI.

D

Create/Edit/Delete

Add/Edit chuyển route sang /new hoặc /:id và render CollectionRecordPage. Delete vẫn chạy trong list bằng ConfirmModal, sau đó reload total và table.

Form động trong CollectionRecordPage

Field metadata/type UI control
options.choices SelectForm; label dạng $t:key được resolve từ Directus /translations.
boolean Checkbox
text, json, csv textarea
date, time, datetime Native date/time input
integer, float, decimal Number input + normalize payload

Route record

PageResolver lấy route khớp dài nhất từ ui_pages, sau đó đọc trailing segments. Với page collection có collection_key, /new render create form và /:id render edit form. Nếu page không có collection_key, segment đầu tiên sau matched route được dùng làm collection key.

API Layer

REST chính

src/lib/utils/apiConfig.ts tạo wrapper apiGet, apiPost, apiPatch, apiDelete. Base URL lấy từ NEXT_PUBLIC_API_HOST, token lấy từ NEXT_PUBLIC_DIRECTUS_TOKEN trong các module REST.

Module Endpoint
ui_pages.api.ts /items/ui_pages
ui_sidebar_items.api.ts /items/ui_sidebar_items
items.api.ts /items/{collection}
fields.api.ts /fields
collections.api.ts /collections
translations.api.ts /translations

GraphQL phụ trợ

src/lib/api/graphql có Apollo client cho /graphql và /graphql/system. Hiện code đã có helpers đọc collections/items, nhưng luồng collection page hiện tại vẫn dùng REST.

Server-side helper

src/lib/api/directus.ts dùng NEXT_PUBLIC_DIRECTUS_URL và DIRECTUS_TOKEN, có directusRequest và getDirectusHealth. API route /api/collections/fields dùng helper này.

State, i18n, Theme

Zustand store

appStore.ts lưu trạng thái sidebar, danh sách Directus fields, collections và translations. Đây là metadata dùng chung cho collection page và dynamic form.

next-intl

Locale lấy từ cookie NEXT_LOCALE, mặc định vi. Message được merge từ common.json, component.json, tenant-service.json.

Theme

AppThemeProvider dùng next-themes với attribute class. Tailwind dark mode dựa trên class dark.

Ghi Chú Hiện Trạng

Điểm cần chú ý khi phát triển tiếp

  • middleware.ts hiện chỉ log pathname và NextResponse.next().
  • PageResolver và một số component còn console.log debug.
  • signin/page.tsx hiện là khung rỗng, chưa dùng SignInForm.
  • Filter đã có component nhưng CollectionPage đang truyền filters rỗng.
  • InputSearch vẫn được import và search state vẫn tồn tại, nhưng phần input search trong CollectionPage đang được comment.
  • AppSidebar.tsx là sidebar mẫu theo shadcn-style, chưa nằm trong layout chính.

Cách thêm page mới

  • Page collection: tạo record ui_pages với page_type=collection và collection_key. List route sẽ render CollectionPage; /new và /:id sẽ render CollectionRecordPage.
  • Page custom: tạo component, export trong customPageRegistry.ts, rồi map bằng component_key.
  • Sidebar: tạo record ui_sidebar_items, set page_id, type, parent_id, sort_order.
  • Nhãn đa ngôn ngữ: ưu tiên translations[locale].title hoặc field meta translations; option label dạng $t:key dùng Directus /translations.
